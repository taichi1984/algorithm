# 📝 エラトステネスの篩 まとめ

## 0.　自分の書いたソースコード
```

/* 素数を求めるプログラム　総当たり
 * エラトステネスの篩で実行する。*/

#include <stdio.h>

int main(void) {

  long n;
  int counter = 0;

  do {
    printf("入力した数値までの素数を表示します。\n "
           "2以上の数値を入力してください :");
    scanf("%ld", &n);
  } while (n < 2);

  char is_prime[n + 1];

  /* フラグ配列を全てtrueで初期化*/

  for (int i = 0; i <= n; i++)
    is_prime[i] = 't';

  for (int i = 2; i * i <= n; i++) {
    if (is_prime[i] == 'f') {
      printf("%d が fのため飛ばす\n", i);
      continue;
    }

    if (is_prime[i] == 't') {
      for (int j = i * i; j <= n; j += i) {
        is_prime[j] = 'f';
        counter++;
      }
    }
  }

  for (int i = 2; i <= n; i++) {
    if (is_prime[i] == 't')
      printf("%-5d  ", i);
  }

  printf("\n");
  printf("除算を行った回数 : %d\n", counter);
  return 0;
}

```
## 1. 基本アイデア

* 「素数の倍数をどんどん消していけば、残った数が素数になる」
* 小さい素数から順に「その倍数を消す」ことを繰り返すだけ。

---

## 2. 内側ループの工夫

```c
for (int j = i * i; j <= n; j += i)
```

* **開始点 j=i\*i**

  * i の倍数のうち、2i,3i,…,(i-1)i はすでに小さい素数で消されている。
  * よって「新しく消す必要があるのは i\*i から」。

* **終了条件 j<=n**

  * 範囲外は調べる必要がない。

* **増分 j+=i**

  * i\*i, (i+1)i, (i+2)i,… と「i の倍数だけ」を飛び飛びで効率的に消せる。

---

## 3. 外側ループの終了条件

```c
for (int i = 2; i * i <= n; i++)
```

* √n まで調べれば十分。
* 理由：

  * n = a × b と割れるとき、必ず **片方は √n 以下**になる。
  * だから √n を超える側は「既に小さい方で消されている」。

例）100000 = 500 × 200

* 200 は √100000≈316 以下 → すでに篩で消されている。

---

## 4. 動作の流れ

1. 配列を「全部素数(true)」として初期化。
2. i=2 からスタート。

   * 素数フラグが立っていれば、その i の倍数を消す。
3. 次の i へ。

   * フラグが false ならスキップ。
4. √n まで繰り返す。
5. 最後に残っているフラグ true の数が素数。

---

## 5. ポイントのまとめ

* **無駄を減らす工夫は3つ**

  1. i\*i からスタート（小さい倍数は既に消えている）
  2. j+=i（効率よく倍数を消す）
  3. √n までで十分（大きい数は小さい相方で消されている）



## 自分が理解に苦しんだ理由
最初に素数を計算して求めるのと混同していた部分があるからかもしれない。
既にfalseにしている以上、そこを飛ばさなければいけないけど、理論上falseになっているはずなのに、もう一度見に行ってtかfかの確認をしようとしていたのが上手くいかなかった理由。
理論上そこは見る必要がないならば、見ないように条件式を組まなければいけない。
